布局方式 - flex布局

Front-end Engineer
  • H5C3

一种相当好用的布局方式 FLEX

  • 我们传统的布局方式其实有很多种，也很好用，比如浮动布局、定位布局之类的
  • 但是传统布局有很大的局限性，对于一些特殊的布局就不是很好实现，比如垂直居中
  • 2009年，W3C 提出了一种新的方案—-Flex 布局，可以简便、完整、响应式地实现各种页面布局。目前，它已经得到了所有浏览器的支持，这意味着，现在就能很安全地使用这项功能。
  • Flex布局将成为未来布局的首选方案

Flex布局的语法介绍

  • 接下来我们就来一个一个的介绍一下 flex 布局中的所有语法
  • 先来准备一个基础解构，接下来如果没有特殊说明，都是使用这个基础解构
  • 这个就是一个基础解构，我们接下来将使用 flex 布局来对这个解构进行各种布局
  • 这个解构就是一个简单的 ul 在页面横向居中，我们主要是来玩里面的 li

开启弹性布局

  • 我们想使用 flex 布局，需要先开启弹性布局，也就是把元素变成弹性元素以后，在进行使用
  • 就好像我们想给一个行内元素设置宽高，需要先把元素变成一个块级元素或者行内块级元素再进行设置是一个道理
  • 开启弹性布局我们使用 display: flex;
  • 上面就是开启弹性布局
  • 这个把元素设置为弹性元素是什么意思呢？
    • 这个开启弹性盒子并不会改变元素自身在标注文档流里面的占位方式
    • 而是改变了当前元素下所有 子级元素 的文档流占位
  • 比如上面这个解构
    • 在开启了弹性以后，没有改变 ul 在文档流里面的占位，ul 该怎么样还是怎么样
    • 而下面所有的 li 都改变了原先的占位方式
  • 在没有开启 flex 的时候，li 应该是从上向下顺序排列
  • 在开启了 flex 以后，li 就会按照 主轴 的方向排列
  • 现在，我们在浏览器里面还看不到效果，是因为当一个元素开启了 flex 以后，会有一些默认的情况出现
    1. 默认主轴的方向是从左到右
    2. 会自动设置所有 子级元素 的 height，但是不会设置 width（如果其子元素自己有设置 height 的话，那么这个默认值就没有用了）
      • 如果其子元素没有设置 height 的话，会自动把所有子元素的高变成自己的 height
    3. 默认所有子元素在一行排列，不会进行换行
    4. 当所有子元素的 width 加起来超过自己本身的 width 以后，不会溢出盒子
      • 默认不会溢出盒子，会把所有子元素的 width 压缩，然他们依旧在自己的范围内，不会溢出
    5. 所染会挤压所有的子元素和 width，但是如果子元素内有内容的话，不会继续挤压
      • 当子元素内有内容的时候，那么这个子元素被挤压的最小的 width 就是内容的 width，不会再小了
      • 当子元素内没有内容的时候，那么就会被无限挤压
  • 说了这么多，我们先来看看效果

  • 开启了 flex 以后，我们会看到，ul 里面的每一个 li 都被自动设置了一个 height，但是没有 width
  • 接下来我们给 li 设置一下 width试试看

  • 我们会发现，每一个 li 都横向排列了（其实这里是按照 ul 的主轴方向）
  • 之前我们不止一次提到过主轴这个词，接下来我们就先看看什么是主轴

  • 当一个元素开启了 flex 以后，会自动出现两个轴
    1. 主轴： 默认是从左到右
    2. 副轴： 默认是从上到下（副轴永远垂直于主轴）
